Notes for Lesson 1: Basics of the Web

The World Wide Web is a collection of html (hypertext markup language) documents. Hyperlinks connect the documents. Major components are a user's computer, the internet, and servers. A browser on the computer allows the user to interact with the internet. HTTP protocol allows the internet to communicate with servers. Here is a video on the subject from the lesson.

HTML is made up of text content (what you see), markup (what it looks like), other documents (e.g. images & videos), and links to other pages. Here is another video on HTML from the lesson.

Markup is made up of "tags"(opening and closing), and "elements" are strings of tagged content. Content includes attributes, and each attribute has a "value." For instance, anchor tags are for making links. Image tags are for displaying images. Images don't have closing tags. A "void tag" has no content. Some tags, such as span, are inline; others, such as div, are block. These are container elements, and their purpose is to contain content inside themeselves. This is yet another video from the lesson, on tags and elements. Create modest html documents with Scratchpad.

The "stupidity" of computers consists in their literal interpretation of commands and data. Unlike human beings, they cannot grasp nuance, nor can they intuit an intention or range of possible intentions behind a programmer's command. This makes even the most simple of programming mistakes (ie, a modest typographical error in an html tag) enough to make an entire process malfunction.

Notes for Lesson 2: Webpages, Documents, and Structure

Webpages are like a house. The structure (html) consists of the foundation and walls, style (css) consists of color and decor, components (javascript) consist of interactive devices (ie, a TV remote or garage door opener) that control things within the structure.

HTML and CSS are both "languages." HTML controls the "structure" of a web page. CSS controls the "style" of a page (how it looks). When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like structure of a page.

Web page structure consists of boxes. Boxification is the process of determining quantity and placement of div elements.

Div elements should be given "class" attributes for the purpose of applying style later. The class attribute should be descriptive and should make sense. The boxification portion of the lesson starts with this video.

Notes for Lesson 3: CSS & the Box Model

CSS stands for "Cascading Style Sheets." It is code written to control the "style" of HTML elements. The "C" is the most important part of this phrase because "Cascading" signifies the function of how style rules will apply to every succeeding element in the DOM unless the rules are overwritten at the element level. The principle that governs this cascading action is "Inheritance," the "mechanism by which properties are applied not ony to a specified element, but also to its descendents" (from the Wikipedia page on CSS). Inheritance contributes to efficiency in programming because it makes it possible to avoid repetition. An html document might contain numerous instances of the same class, and it would be prohibitively time-consuming to have to stylize each separately. Cascading also optimizes efficiency with page-loading time and bandwidth.

The "selector" in CSS corresponds to named tags (such as an "h1" font identifier, or a "description" class id) in the html and governs their appearance. Add style with an attribute, followed by a colon, followed by a value. Declarations, properties, and values exist within the curly brackets {} in the CSS element and act as a rule for every instance of the selector in the html DOM. See Andy's demos for good examples.

The Box Model: margin, border, padding, content. Content is the nucleus, protected by the rest of the elements to prevent other content from encroaching on it. Padding clears the area around the content and is affected by the background color of the box. It is the space inside the box that protects the content directly. Border goes around padding. The outer layer is margin, and has no background color. Box sizes can be specified by pixels or percentages. The process of "Code, Test, and Refine" can be broken down into the following steps for front-end development: 1) Look for natural boxes; 2) Look for repeated styles & semantic elements; 3) Write your html; 4) Apply styles (from Biggest to smallest); and 5) Fix things.
